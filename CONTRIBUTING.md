# Contributing to SharePad

First off, thank you for considering contributing to SharePad ❤️

Contributions of all kinds are welcome — bug fixes, feature improvements, documentation updates, UI/UX enhancements, and general feedback.

## Before You Start

If you've found a bug or want to suggest a feature, please first check the existing [issues](https://github.com/pratham-jaiswal/sharepad/issues) to avoid duplicates.

If a similar issue does not already exist, feel free to open a new one with as much detail as possible.

For larger changes or new features, opening an issue for discussion before starting work is recommended.

---

## Fork & Create a Branch

Fork the repository and create a descriptive branch for your work.

Example (where issue `#107` is the issue being worked on):

```bash
git checkout -b feature/107-add-socketio
```

Other examples:

```bash
git checkout -b fix/142-markdown-preview-scroll
git checkout -b docs/update-readme
```

---

## Development Guidelines

- Keep changes focused and minimal where possible.
- Follow the existing code style and project structure.
- Test your changes before submitting a pull request.
- Update documentation if necessary.

---

## Pull Requests

When submitting a pull request:

- Clearly describe the changes made.
- Reference related issues when applicable.
- Include screenshots/videos for UI changes if relevant.

Please keep pull requests focused and avoid unrelated changes in the same PR.
